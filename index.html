Jump Game

发表于 2020-02-19

Jump Game
Given an array of non-negative integers, you are initially positioned at the first index of the array.
Each element in the array represents your maximum jump length at that position.
Determine if you are able to reach the last index.

DP问题，比较喜欢用top-down解决。
top-down和bottom-up都需要维持一个数据结构记录子问题的结果。

第一个答案存在的问题：

  1. 没有区分unvisited和false，导致false时无法直接利用子问题的解。可以把canJumpFrom存储的值设为true/false/unvisited.
  2. canJumpFrom数组存储了结果，canJump没有必要再每次return结果

其他方法

Greedy

canReachLast = last index，从后last index开始往前，找到第一个能跳到canReachLast的，设为新的canReachLast（每次能跳到canReachLast的也一定能跳到最后），直到canReachLast==0
Time complexity : O(n)
Space complexity : O(1)
（草，这个方法好厉害）

Largest Rectangle in Histogram~为什么用单调栈

发表于 2020-02-06 更新于 2020-02-07

Largest Rectangle in Histogram
Given n non-negative integers representing the histogram’s bar height where the width of each bar is 1, find the area of largest rectangle in the histogram.

在直方图里找到一个最大的长方形，首先一个很直观的方法就是找到所有可能的长方形，然后选出最大的一个。但有一些长方形很明显不会是最大的（比如没有紧贴着边缘的那些），所以我们要尽量减少面积的计算，只算那些”足够大”的长方形。
如果我们从前往后遍历每根柱子，每次都向前尽量“撑大”，不难想到“足够大”的长方形应该是这些：

然后我们发现，这些长方形中还是有可以省略的，比如(a)(b)中蓝色和绿色的部分，在(C)中都会被包括起来。另外，前面那些比较高的柱子对于之后的计算似乎没什么影响。
从前往后遍历时，注意到两点：

  • 当前柱子向前计算时，没必要算那些“特别矮”的（以后会有人收拾他的）
  • 当前柱子向前计算时，那些比它高的柱子没什么用

这就是使用单调栈的意义，每次入栈时，让比我高的（没用的）柱子计算一下然后出栈，保证我进来的时候前面的柱子都比我矮，我出栈的时候计算也会很方便。

总而言之，一步步减少计算量的思路如下：
计算所有长方形 -> 遍历每根柱子向两边扩张 -> 只向前扩张 -> 只向前扩张足够高的柱子（矮柱子之后会有人做）-> 忽略前面那些比自己高的柱子（对面积无影响）

所以这道题算不算动态规划呢？DP的条件是有重叠子问题和最优子结构，主要是这个算不算是重叠子问题呢？

还看到了一种方法，就是遍历每一个柱子时，以这个柱子为高度标准向两边扩张。实现了一下发现效率不如单调栈，因为中间有很多重复的比较。（好像也可以优化，但是我不想看这道题了…）

NQueens~回溯法

发表于 2020-01-18 更新于 2020-01-19

NQueens
The n-queens puzzle is the problem of placing n queens on an n×n chessboard such that no two queens attack each other.

Given an integer n, return all distinct solutions to the n-queens puzzle.

Each solution contains a distinct board configuration of the n-queens’ placement, where ‘Q’ and ‘.’ both indicate a queen and an empty space respectively.

回溯法（backtracking）是暴力搜索法（Brute-force search）的一种。基于试错的思想，为达到目标一步步向前探索，发现走不通则向后退一步重新选择路径，直到找到所有解。回溯通常用递归实现。

回溯法能找到问题的所有解，因此适用于n皇后问题。

比较需要注意的地方是,递归调用回溯函数时，在退出函数的地方要把相关数据还原到进入函数前的状态。
backtracking属于dfs的一种应用，不保留完整的树结构，遇到不能再走下去的路径是直接“剪枝”。

NQueens 2
Given an integer n, return the number of distinct solutions to the n-queens puzzle.

Java字符串拼接的性能比较

我不记得字符串拼接最快的方法，但我记得直接+++的方法是最慢的方法。
复习了一下字符串拼接常用的四种方法，并测试了它们的性能：

  1. “a”+”b”+”c”
  1. String concat方法

  1. StringBuffer

  2. StringBuilder

StringBuffer和StringBuilder都是比较高效的，它们有什么区别呢？
StringBuffer是Synchronized，线程安全的；
StringBuilder是非线程安全的，适用于单线程的情况。

总结：操作少量数据是可以用String，单线程大量数据时用StringBuilder，多线程大量数据时用StringBuffer.
