Hexo+Typora+GitHub~提高写博客体验workflow

发表于 2020-04-18 更新于 2020-04-19 分类于 其他工具

之前有一段时间，不管写前端写Java写Python还是写博客都是vscode，个人感觉vscode+丰富的插件真的相当万能，但万能既是优点也是缺点。对于写Markdown记笔记博客来说，vscode（以及其他IDE）比较不方便的地方在于：

  • 不支持所见即所得。我感觉所见即所得对md记笔记来说还是很重要的，尤其是有图片的地方。我之前一直半边开md显示插件，但这样做相当于界面的内容一半是重复的，很占地方。
  • 插入图片麻烦。不知道是不是有更高效的方法，但我之前是先把要用的图片复制到images文件夹，为了整齐再给图片改个名字，再在Markdown里写上路径😂。。

总之，受够在vscode里写Markdown了，决定今天好好整理一下workflow。相信好的写作体验会让我写更多博客（x

Typora

Typora是一款支持实时预览的Markdown文本编辑器，官网说它是 / A truly minimal markdown editor. /

默认使用和GitHub相同的Markdown语法规范：GFM标准，在偏好设置里还可以自定义语法偏好。

GitHub设置

前几天git config global配置成了研究室Bigbucket用的账号，今天试着直接更新了博客，发现竟然可以更新，但GitHub上的commit信息是研究室账号（原来只要连了SSH谁都可以更新我的仓库，害怕😳）怪不得实习的时候GitHub要开双重验证。

git可以在local仓库配置用户:

git config user.name "username"

git config user.email "your email"

可以在随便什么地方配置全局用户:

git config --global user.name "username"

git config --global user.email "your email"

我就想着以后GitHub的仓库单独配置一下local，global还是用研究室账号，但是发现好像Hexo的git我不能显式设置（也可能是我没找到），所以最后还是选择global设置为GitHub账号的方案。

往GitHub上传代码时，GitHub根据email判断你是谁（和user.name无关），没有在GitHub注册的email也可以push，只是提交记录只会显示一个灰灰的头像👇
